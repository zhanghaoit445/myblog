## 项目管理

### 什么是项目

项目(project)指的是 **为创造独立性产品,服务或成果所展开的有期限约束的业务**

主要特点为:

1. 有期限约束,必须在规定时间内完成,当然项目也是可以延期的,知名的跳票大王比如暴雪呀,暴雪呀,
还有暴雪呀,他们会为了项目质量而增加开发周期.
2. 独立性,项目不是周期性的劳作,每一个项目都必然有其独特性,比如银行中的存款,取款业务就不是项目;
而投资就是项目,因为投资需要对被投资者进行审查,而每个投资的法人都是不同的.


项目业务分类:

分类|有无明确起始截止日期|是否独特|例子
---|---|---|---
项目|有|是|新手机开发,举办奥运会,员工旅行,婚礼,投行投资,银行放贷
季节性业务|有|否|年末大扫除,端午包粽子,秋收
发明等|无|是|发明灯泡,发明计算机
固定业务|无|否|工厂生产汽车,每日打扫,银行柜台存取款


### 什么是项目管理

项目管理:是指为确保项目顺利实现目标而采取的行为就是项目管理

项目管理的知识体系模块(并非每个项目都全部包括):

+ 沟通管理
+ 领域管理: 项目任务的决定方法,必要工作的筛选及进展方式等
    + 品质管理:
        + 时间管理:确保具有期限性的项目能够按时完成而进行的日程规划和管理
        + 成本管理
        + 人力资源管理
        + 采购管理
    + 风险管理


### 项目的目的与成功标准

项目的目的也就是目标以及成功标准是需要明确定义的,如果可以的话最好使用可以量化的目标来作为标准

比如一个游戏项目的目标不应定为'大受好评',而应该是比如卖出10w份,获得fami通白金评价等

目的的制定应该根据对应的场景和条件,带有明确的目的性,而成功标准则必须明确,最好可量化,这样便可以针对标准展开行动



### 项目管理者的职责

项目管理者的职责主要是:

+ 对制约条件的合理安排,如日程,预算,这些都是制约项目的条件,巧妇难为无米之炊,
如何对现有资源给予合理的安排和调度,让资源最大化利用,同时又能够通过沟通让各个部门不会因为
资源问题影响积极性是项目管理者的首要问题

+ 计划进展支持调整,为项目制定目标,计划,追踪计划,跟进计划,根据外部条件和项目进度及时调整计划

总的来看项目管理者做的是个保姆的活,他的主要职责是沟通


### 什么是利益相关者

利益相关者是指与项目直接相关的人

利益相关者包括:

1. 直接相关者:
    + 项目团队成员
2. 受影响者:
    + 项目业务的受众
    + 项目业务的相关从业人员

比如说,一个修建公路的项目,它的利益相关者有,团队成员:设计单位,施工单位;项目业务受众:周边居民,
司机;相关从业人员:包括材料商,当地政府等


## 项目策划

项目策划的主要工作就是制作项目规划案.

项目规划案有三大主要目的:

+ 分析可行性:需要将计划都写下来进行一一判断,查漏补缺,看整体是否存在矛盾
+ 与利益相关者共有(共同参与):让参与者都统一目标不会有理解上的偏差,一来减少概念模糊,二来也可以在产生倦怠时回顾,以此振奋士气
+ 项目进展过程中对照使用:项目规划案不可能一劳永逸,需要定期将工作进度和计划进行对照,确认差异,修改计划

项目计划案在初版中,如果有很多不确定不明白的地方需要写出来,并把"知道的内容写上",
知道多少写多少.对于无法纳入计划的,可以写在风险或课题部分,等到具体实施的时候再讨论.
这种不在最初阶段决定所有计划细节,分步详细化的过程叫"阶段性详细化".

项目规划案内容包括:

+ 项目目标:制定目标需要结合自身情况和收集的利益诉求来制定,目标必须有明确成功标准

+ 范围定义:明确实现目标所需任务

+ 制作WBS: 工作安排,任务细化(Work Breakdown Structure),为使任务与工作便于管理,将项目案一定原则进行分解具体分为:
    + 范围内的任务(主要)
    + 外围事项
    + 制约条件:时间,预算,所需的资源和工具
    + 前提条件

    WBS可以有多层,逐层细化,直到分解致项目管理工作的单位--活动

    WBS三项原则:
    + 100%原则:WBS下一级别的分支为上一级别要素的所有工作内容.
    + WBS1级为项目整体
    + WBS2级记录项目范围和项目管理要素

+ 活动定义:明确完成任务所需工作

+ 活动顺序设定:明确完成任务的工作顺序

+ 估计必需的资源及时间

+ 制作日程表:制作实现目标的工作日程表

+ 估计成本:计算实现目标的必要成本

+ 决定其他规划:讨论采购,品质,风险,体制等
